## Folder Policy

- `docs/concepts/`
  - vision, roadmap, planning, priorities, and phased delivery notes
- `docs/design/`
  - technical specifications, behavior models, configuration grammar, and
    operational design
- `docs/references/`
  - local copies of primary source material

## Working Rule

All new project documents should have:

- an English version
- a Japanese version
- placement in either `concepts` or `design`, never mixed casually
